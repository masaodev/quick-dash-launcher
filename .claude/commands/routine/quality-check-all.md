# Quality Check All

quality-checkerエージェントを使用して、コード品質を総合的にチェックします。

## 実行内容

このコマンドは `quality-checker` エージェントを呼び出し、以下のチェックを自動実行します：

### チェック項目

1. **TypeScript型チェック**
   - `npm run type-check` の実行
   - 型エラーの検出と報告

2. **ESLintチェック**
   - `npm run lint` の実行
   - コード規約違反の検出

3. **コード複雑度**
   - 循環的複雑度の高い関数の特定
   - ネストが深い箇所の検出

4. **命名規則**
   - 変数・関数・クラス名の一貫性チェック
   - TypeScript/React命名規則への準拠確認

5. **重複コード**
   - 類似コードブロックの検出
   - リファクタリング候補の提示

6. **ドキュメント不足**
   - JSDoc/コメント不足の検出
   - 複雑な関数への説明追加推奨

7. **テストカバレッジ**
   - 未テストコードの特定
   - カバレッジ率の確認

8. **ファイル構造**
   - 大きすぎるファイルの検出（500行超）
   - 適切な分割提案

9. **インポートパス**
   - 相対パス/絶対パスの一貫性チェック
   - エイリアス使用の推奨

10. **セキュリティリスク**
    - 潜在的なセキュリティ問題の検出
    - 危険なパターンの警告

## 実行方法

```
/quality-check-all
```

内部的に以下のエージェントが実行されます：

```
Task tool with subagent_type="quality-checker"
```

## 出力形式

エージェントは以下の形式でレポートを生成します：

- **Critical**: 即座に対応が必要な問題
- **High**: 早急な対応が推奨される問題
- **Medium**: 改善が望ましい箇所
- **Low**: 軽微な改善提案

## 使用方法

```
/quality-check-all
```

## 実行タイミング

- コミット・PR作成前（`commit-pr-full`で自動実行）
- リファクタリング前
- 週1回の定期メンテナンス
- コードレビュー前

## 注意事項

- 実行には数分かかる場合があります
- Critical問題が検出された場合は、コミット前に必ず修正してください
- quality-checkerエージェントの詳細は `.claude/agents/quality-checker.md` を参照
