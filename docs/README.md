# ドキュメント構成ガイド

このディレクトリには、QuickDashLauncherプロジェクトの各種ドキュメントが格納されています。

## ディレクトリ構造

```
docs/
├── guides/              # 開発者向けガイド
├── manual/              # ユーザーマニュアル
├── reference/           # 仕様書・リファレンス
│   ├── architecture/    # アーキテクチャ
│   └── screens/         # 画面仕様書
├── testing/             # テスト関連ドキュメント
└── templates/           # テンプレート
```

## 各ディレクトリの役割

### 📖 guides/ - 開発者向けガイド

**目的**: プロジェクトの開発方法を説明する開発者向けガイドドキュメント

**対象読者**: 開発者（コントリビューター、メンテナー）

**記載内容**:
- プロジェクトのセットアップ方法
- 開発の進め方・実装詳細
- ビルド・デプロイ手順
- テスト手順
- コード品質ガイドライン
- Git/GitHubワークフロー

**例**:
- `getting-started.md` - プロジェクトのセットアップ
- `development.md` - 開発ガイドと実装詳細
- `build-and-deploy.md` - ビルドとデプロイ
- `manual-checklist.md` - テストチェックリスト（testing/ディレクトリに移動）
- `project-overview.md` - プロジェクト概要
- `github-workflow.md` - Git操作とPRフロー

---

### 📖 manual/ - ユーザーマニュアル

**目的**: 機能の使い方を説明するエンドユーザー向けマニュアル（「どう使うか」に焦点）

**対象読者**: エンドユーザー（アプリケーション利用者）

**記載しないもの**:
- データ形式の完全仕様 → `reference/`へ
- コーディングの実装詳細 → `reference/architecture/`へ

**例**:
- `icon-system.md` - アイコンシステムの概要
- `group-launch.md` - グループ起動の使い方
- `folder-import-item.md` - フォルダ取込の使い方
- `item-management.md` - アイテム管理画面の操作方法

### 📚 reference/ - 仕様書・リファレンス

**目的**: 機能の仕様を体系的にまとめた参照資料（「何ができるか」「どう書くか」に焦点）

**対象読者**: ユーザー・開発者

**特徴**:
- ✅ 仕様のみを記載（構文・オプション・動作）
- ✅ 網羅的・完全な情報
- ✅ 辞書的に使う（必要な時に引く）

**記載内容**:
- データファイル形式仕様
  - フィールド構成、構文、オプション一覧
- アプリケーション設定仕様
  - 設定項目、デフォルト値、範囲
- アイコン処理仕様
  - 保存先、取得順序、対応形式
- キーボードショートカット仕様
- 画面仕様書（`screens/`サブディレクトリ）

**記載しないもの**:
- ❌ 使い方の手順 → `manual/`へ
- ❌ ユースケース・実例 → `manual/`へ

**例**:
- `data-file-format.md` - データファイル形式仕様
- `settings-specification.md` - アプリケーション設定仕様
- `icon-specification.md` - アイコン処理仕様
- `keyboard-shortcuts.md` - キーボードショートカット仕様
- `screens/screen-list.md` - 画面仕様書一覧

## ディレクトリの使い分け

### manual/ と reference/ の違い

| 観点 | manual/ | reference/ |
|------|-----------|------------|
| **目的** | 使い方を説明 | 仕様を列挙 |
| **焦点** | 「どう使うか」 | 「何ができるか」「どう書くか」 |
| **読み方** | 順を追って読む | 辞書的に引く |
| **内容** | ユースケース、手順、実例 | 構文、オプション、完全仕様 |

**判断例**:

| 内容 | 置くべき場所 |
|------|-------------|
| グループ機能の使い方と実例 | **manual/** |
| グループ構文の完全仕様 | **reference/** |
| `depth`オプションの全値と説明 | **reference/** |
| 「開発環境を一括起動する例」 | **manual/** |
| UI操作手順（Ctrl+Eで開く...） | **manual/** |
| data.txtのフィールド定義 | **reference/** |

## ドキュメント作成ガイドライン

### 1. ファイル命名規則

- 小文字とハイフンを使用: `feature-name.md`
- 英語での命名を推奨（日本語コンテンツでもOK）

### 2. 推奨サイズ

- 1ファイル: **100〜200行を目安**
- 大きくなりすぎた場合は分割を検討

### 3. 相互リンク

- 関連ドキュメントへのリンクを明記
- 相対パスを使用

### 4. 更新ルール

- ドキュメントは常に最新の状態に保つ
- 機能追加・変更時は関連ドキュメントも更新
- 古い情報は削除または更新
