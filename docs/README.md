# ドキュメント構成ガイド

このディレクトリには、QuickDashLauncherプロジェクトの各種ドキュメントが格納されています。

## ディレクトリ構造

```
docs/
├── guides/              # ガイド類
├── features/            # 機能詳細
├── architecture/        # アーキテクチャ
├── reference/           # リファレンス情報
│   └── screens/         # 画面仕様
├── meta/                # メタ情報
└── templates/           # テンプレート
```

## 各ディレクトリの役割

### 📖 guides/ - ガイド類

**目的**: プロジェクトの使い方や開発方法を説明するガイドドキュメント

**記載内容**:
- プロジェクトのセットアップ方法
- 開発の進め方
- ビルド・デプロイ手順
- テスト手順

**例**:
- `getting-started.md` - 初めてのセットアップ
- `development.md` - 開発ガイド
- `build-and-deploy.md` - ビルドとデプロイ
- `testing.md` - テストチェックリスト

---

### ⚙️ features/ - 機能詳細

**目的**: アプリケーションの個別機能の詳細仕様を記載

**記載内容**:
- 機能の概要と目的
- 使用方法
- 設定オプション
- 実装の詳細

**例**:
- `icon-system.md` - アイコン取得・管理システム
- `group-launch.md` - グループ起動機能
- `folder-import-item.md` - フォルダ取込機能
- `item-management.md` - アイテム管理機能

---

### 🏗️ architecture/ - アーキテクチャ

**目的**: システムの設計・構造を説明する技術ドキュメント

**記載内容**:
- システム全体の構造
- プロセス間通信
- データフロー
- コンポーネント設計

**例**:
- `overview.md` - システム概要
- `ipc-channels.md` - IPC通信仕様
- `data-flow.md` - データ処理フロー
- `window-control.md` - ウィンドウ管理

---

### 📚 reference/ - リファレンス情報

**目的**: 機能的な事実を辞書的に引くための参照資料

**記載内容**:
- データ形式仕様（ユーザーが編集するファイル形式）
- キーボードショートカット一覧
- 画面一覧（`screens/`サブディレクトリ）
- コマンド・オプション一覧

**記載しないもの**:
- コーディングの詳細（関数名、クラス名、実装ロジック）
- 内部API仕様
- 開発者向けの技術詳細

**例**:
- `data-file-format.md` - data.txtの形式仕様
- `keyboard-shortcuts.md` - キーボードショートカット
- `screens/screen-list.md` - 画面一覧

#### 📱 reference/screens/ - 画面仕様

**目的**: UI画面の詳細仕様書

**記載内容**:
- 画面の構成
- 表示条件
- ユーザーアクション
- 画面遷移

**例**:
- `screen-list.md` - 画面一覧（インデックス）
- `main-window.md` - メインウィンドウ仕様
- `bookmark-import-modal.md` - ブックマークインポート仕様

---

### 📝 meta/ - メタ情報

**目的**: ドキュメント管理やプロジェクト運営に関する情報

**記載内容**:
- ドキュメント管理ルール
- コードレビューレポート

- 開発プロセス

**例**:
- `documentation-rules.md` - ドキュメント管理ルール

---

### 📄 templates/ - テンプレート

**目的**: 新規ドキュメント作成時のテンプレート

**記載内容**:
- 画面仕様書のテンプレート

## ドキュメント作成ガイドライン

### 1. ファイル命名規則

- 小文字とハイフンを使用: `feature-name.md`
- 英語での命名を推奨（日本語コンテンツでもOK）

### 2. 推奨サイズ

- 1ファイル: **100〜200行を目安**
- 大きくなりすぎた場合は分割を検討

### 3. 相互リンク

- 関連ドキュメントへのリンクを明記
- 相対パスを使用

### 4. 更新ルール

詳細は [ドキュメント管理ルール](meta/documentation-rules.md) を参照してください。
