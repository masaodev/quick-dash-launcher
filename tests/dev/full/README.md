# テストデータセット - Full

このディレクトリには、QuickDashLauncherの全機能を網羅したテストデータが含まれています。

## 起動方法

```bash
npm run dev:test
```

## 設定内容

### ホットキー
- **Ctrl+Alt+T** - QuickDashLauncherを表示/非表示

### Viteポート
- **9003** - 開発サーバーポート（devやdev2と重複しない）

### ウィンドウサイズ
- メインウィンドウ: 700x500
- 編集モード: 1200x700

## データファイル構成

データファイルは `data.txt`、`data2.txt`、`data3.txt` のように自動採番されます。
ファイル名は変更できず、アプリケーションが自動的に管理します。

### 1. data.txt（メイン）
全機能を含む充実したサンプルデータ：

- **Webサイト**: GitHub、Google、YouTube、Zenn
- **ツール**: ChatGPT、Claude、Google翻訳
- **ローカルフォルダ**: デスクトップ、ダウンロード、ドキュメント、ピクチャ
- **アプリケーション**: メモ帳、電卓、ペイント
- **カスタムURIスキーム**: VS Code、PowerShell設定
- **グループアイテム**: 開発環境、調べ物、デザイン（複数アイテム一括起動）
- **ワークスペース**: ワークスペース管理画面
- **ウィンドウ操作**: メモ帳（中央配置）、電卓（左上配置）

### 2. data2.txt（仕事）
仕事用のアイテム：

- プロジェクト管理: GitHub Projects、Notion
- コミュニケーション: Slack、Teams
- ドキュメント: Google Docs、Sheets
- 開発セット（グループ）
- プロジェクトフォルダ

### 3. data3.txt（プライベート）
プライベート用のアイテム：

- エンタメ: YouTube Music、Netflix、Amazon Prime
- SNS: Twitter、Instagram
- ショッピング: Amazon、楽天市場
- 生活セット（グループ）
- 個人フォルダ

## マルチタブ機能

このテストデータでは、マルチタブ機能が有効化されています：

- **メイン**: data.txt（全機能のデモ）
- **仕事**: data2.txt（仕事関連アイテム）
- **プライベート**: data3.txt（プライベートアイテム）

タブ切り替えでコンテキストに応じたアイテムにすばやくアクセスできます。

## テスト可能な機能

このテストデータで以下の機能を確認できます：

- ✅ Webサイト起動
- ✅ ローカルアプリケーション起動
- ✅ フォルダ/ファイル操作
- ✅ カスタムURIスキーム（vscode://、ms-settings:等）
- ✅ グループアイテム（複数アイテム一括起動）
- ✅ ワークスペース管理
- ✅ ウィンドウ操作（位置・サイズ調整）
- ✅ マルチタブ切り替え
- ✅ 検索機能
- ✅ 編集モード

## Playwright MCPでの利用

`npm run dev:test`で起動すると、リモートデバッグポート（9222）が有効化され、Claude Codeから操作可能になります。

```bash
# 起動
npm run dev:test

# Claude Codeから操作例
「QuickDashLauncherのスクリーンショットを撮ってください」
「検索ボックスに『GitHub』と入力してください」
「メインタブをクリックしてください」
```

詳細な操作方法は **[ブラウザ自動操作ガイド](../../../docs/testing/browser-automation.md)** を参照してください。

## 注意事項

- このテストデータはGit管理されており、チーム全体で共有されます
- 個人の設定で上書きされることはありません（`QUICK_DASH_CONFIG_DIR`で分離）
- バックアップ機能は無効化されています（テストデータの変更を防ぐため）
